## L’objectif (éviter le “tool zoo”)

L’objectif n’est pas d’utiliser plus d’outils. L’objectif est de réduire la friction et publier de façon régulière :

- un workflow hebdo répétable
- un “quality bar” atteignable
- une boucle de mesure (quoi améliorer, quand)

## Le workflow en 7 étapes (la colonne vertébrale)

1. **Brief** : audience, offre, preuve, CTA, contraintes compliance.
2. **Script** : hook → preuve → démo → CTA. Une idée par vidéo.
3. **Plans** : 6–12 plans pour une pub 30–45s (ou 3–6 pour un short 15s).
4. **Assets** : b‑roll généré, visuels produit, logos, captures.
5. **Assemblage** : draft template + rythme + structure.
6. **Finition** : captions, son, exports, QA.
7. **Publication & itération** : mesurer, garder les gagnants, couper le reste.

Si une étape manque, la qualité devient aléatoire. (La plupart sautent “plans” et “finition”.)

## Map outils (qui fait quoi)

Vous pouvez mixer, mais une creator stack ressemble souvent à :

- **B‑roll / plans concepts :** <a href="/fr/tools/kling/">Kling</a>
- **Drafts templates & variantes :** <a href="/fr/tools/invideo/">InVideo</a>
- **Captions + polish final :** <a href="/fr/tools/capcut/">CapCut</a>
- **Voiceover :** <a href="/fr/tools/elevenlabs/">ElevenLabs</a> (selon règles)
- **Avatars + localisation :** <a href="/fr/tools/heygen/">HeyGen</a> et <a href="/fr/tools/akool/">Akool</a>
- **Automatisation (handoffs, logs, checklists) :** <a href="/fr/tools/make/">Make</a>
- **Business :** <a href="/fr/tools/systeme/">Système.io</a>, <a href="/fr/tools/shopify/">Shopify</a>, <a href="/fr/tools/wordpress/">WordPress</a>

## Structure de fichiers & naming (petit effort, gros ROI)

Si vous produisez des variantes, le naming vous sauve. Exemple simple :

```
2026-04_nom-campagne/
  00-brief/
  01-scripts/
  02-plans/
  03-assets/
  04-montages/
  05-exports/
  06-resultats/
```

Et nommez les exports pour pouvoir les lire dans un tableur :

`plateforme_angle_hook_preuve_cta_v01.mp4`

## Checklist qualité (avant publication)

**Créa**

- Hook visible et spécifique dès les 2 premières secondes.
- Une idée centrale (pas de “feature dump”).
- Preuve réelle (démo, chiffres, social proof, claims cadrés).
- Captions lisibles sur mobile (marges, contraste, taille).

**Technique**

- Niveaux audio cohérents (la voix passe avant tout).
- Export 9:16 propre (rien de coupé).
- Première frame intentionnelle (miniature).

**Risque / compliance**

- Pas d’usurpation trompeuse / deepfake “mensonger”.
- Claims encadrés (résultats variables, conditions).
- Assets licenciés + consentement pour visages/voix.

## La boucle de mesure (hebdo, pas quotidien)

Choisissez quelques métriques, et améliorez une chose par semaine :

- **Hook rate / thumbstop**
- **Rétention** (où ça décroche)
- **CTR**
- **CVR** (landing)

La stack sert à rendre cette boucle “cheap”.

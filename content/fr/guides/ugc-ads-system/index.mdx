## Ce que ce système optimise

- **Vitesse d’apprentissage** (trouver ce qui marche vite)
- **Régularité** (publier chaque semaine sans burn-out)
- **Contrôle** (savoir quelle variable change la perf)
- **Sécurité** (claims maîtrisés, pas d’usurpation trompeuse)

## Anatomie en 4 blocs (simple et scalable)

Pour des pubs 9:16, cette structure est très solide :

1. **0–3s Hook** : une promesse ou une douleur (captionée).
2. **3–10s Preuve** : crédibilité rapide (démo, social proof, claim encadré).
3. **10–20s Démo** : montrer le “comment” avec rythme.
4. **20–30s CTA** : une offre, une action.

## Matrice de test (éviter le chaos)

Partez d’une offre + une audience. Puis générez des variantes via :

- **3 hooks** (douleur, résultat, curiosité)
- **2 preuves** (démo vs social proof)
- **2 CTA** (soft vs direct)

Soit **12 variantes**. Vous pouvez démarrer à **6** (3 hooks × 2 CTA) si budget serré.

Règle : **une variable à la fois** quand vous cherchez un apprentissage propre.

## Workflow de production (agnostique outils)

1. Script base + 3 hooks.
2. 6–12 assets : clips produit, captures, inserts b‑roll.
3. Drafts dans un template editor (<a href="/fr/tools/invideo/">InVideo</a> est fait pour ça).
4. Finition dans un éditeur (<a href="/fr/tools/capcut/">CapCut</a>) : captions + rythme.
5. VO si nécessaire (<a href="/fr/tools/elevenlabs/">ElevenLabs</a>), courte et claire.
6. Exporter + nommer pour tracking.

Pour du b‑roll généré : <a href="/fr/tools/kling/">Kling</a>. Pour présentateur/localisation : <a href="/fr/tools/heygen/">HeyGen</a> / <a href="/fr/tools/akool/">Akool</a>.

## Les métriques qui comptent

- **Thumbstop / 2s hold**
- **Rétention**
- **CTR**
- **CPA / ROAS**

Hook faible = problème de promesse. Drop mid-video = rythme/clarté. CTR faible = offre/preuve.

## Échecs fréquents (et fixes)

- **Trop de claims** : une promesse + une preuve.
- **Captions génériques** : concret (“Gagne 2h/semaine”, pas “Boost productivité”).
- **Pas de démo** : montrer vite.
- **Over-editing** : effets = baisse de confiance dans beaucoup de niches.

## Compliance & brand safety (réalité 2026)

- Éviter l’usurpation trompeuse / faux porte-parole.
- Utiliser voix/visages consentis + assets licenciés.
- Claims vérifiables + texte clarifiant si besoin (“résultats variables”, “conditions”).
- Quand doute : transparence > “astuce”.

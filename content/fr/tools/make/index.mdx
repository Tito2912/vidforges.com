## C’est quoi Make ?

Make est une plateforme d’automatisation visuelle. Vous créez des **scénarios** qui connectent des applications et s’exécutent :

- sur événement (nouveau lead, nouvelle commande, nouveau fichier)
- sur planning (toutes les heures/jours)
- via webhook (temps réel)

Pour un site d’affiliation / contenu, c’est idéal pour automatiser la collecte de leads, le reporting, et l’organisation des assets.

## Idéal pour

- Routage des leads (formulaire → CRM → email)
- Ops contenu (checklists, alertes, mise à jour de tableurs)
- Ops e‑commerce (commande → support → logistique)
- Reporting (métriques quotidiennes)

## Tarifs (vérifiés avril 2026)

Make facture surtout selon les **crédits** (chaque action de module compte généralement comme un crédit). En avril 2026, la page officielle indique notamment :

- **Free** : $0/mois (jusqu’à 1 000 crédits/mois)
- **Core** : $9/mois (prix affiché pour 10k crédits/mois)
- **Pro** : $16/mois (prix affiché pour 10k crédits/mois)
- **Teams** : $29/mois (prix affiché pour 10k crédits/mois)
- **Enterprise** : sur devis

En pratique, votre coût dépend de **crédits/jour × scénarios**. Mesurez 7 jours de runs (et la conso réelle) avant de choisir un plan.

## Concepts clés

- **Trigger** : déclencheur
- **Modules** : étapes (chercher, transformer, créer, envoyer)
- **Opérations** : mesure de consommation selon plans
- **Routes** : branchements conditionnels
- **Gestion d’erreurs** : retries, fallback, alertes

## Scénario de départ (stack créateur)

Exemple : “Nouveau lead → email de bienvenue → alerte Slack → ligne Google Sheets”

1. Trigger : nouvelle soumission (ou lead Système.io).
2. Validation : nettoyer/formatter les champs.
3. Action : email de bienvenue (ou ajout à votre outil email).
4. Action : notification Slack/Discord.
5. Log : ajout dans Google Sheets.

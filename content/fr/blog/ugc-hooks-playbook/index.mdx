## La règle : tester les hooks d’abord (avant le reste)

Si le hook est faible, le montage ne sauvera pas la perf. On corrige d’abord les 3 premières secondes :

- nouveau texte
- nouveau plan d’ouverture
- nouvelle promesse/preuve

Pour le système pilier complet (structure d’A/B, naming, cadence), lisez : <a href="/fr/guides/ugc-ads-system/">Système UGC Ads</a>.

## Catégories de hooks (pour éviter de tourner en rond)

| Catégorie | Pattern | Exemple (template) |
|---|---|---|
| Douleur → solution | Problème direct | “Si vous faites encore **X**, vous perdez **Y**…” |
| Vérité inattendue | Casser une croyance | “Tout le monde dit **X**. Faux. Voilà pourquoi…” |
| Proof-first | Preuve dès le début | “J’ai tracké ça 7 jours… voici le résultat.” |
| Demo-first | Action immédiate | “Regardez ça—la façon la plus rapide de…” |
| Erreur | Éduquer sans juger | “Arrêtez **X**—faites **Y**.” |
| Comparaison | A vs B | “J’ai testé **A vs B**… le gagnant m’a surpris.” |
| Offre | Deal upfront | “Aujourd’hui : **-X%** + livraison offerte.” |
| Histoire | Mini narratif | “J’allais abandonner… puis j’ai testé ça.” |

## Captions & texte à l’écran (pour que le hook atterrisse)

Les captions font partie du hook. Règles simples :

- une idée par ligne
- fort contraste, safe margins, pas de texte minuscule
- garder le hook à l’écran 1–2s de plus que prévu
- même offre partout (audio + texte)

Stack :

- captions + finition : <a href="/fr/tools/capcut/">CapCut</a>
- drafts templates et variantes : <a href="/fr/tools/invideo/">InVideo</a>
- b‑roll concepts : <a href="/fr/tools/kling/">Kling</a>

## Système de tests propre (pour apprendre vraiment)

1. Même offre, même proof, même CTA, même longueur.
2. Changer **une seule chose** : les 1–3 premières secondes.
3. Lancer 3–5 hooks par angle.
4. Choisir un winner, puis tester **proof** ou **CTA**.

Matrice simple :

```
Angle : "Gagner du temps"
Hook A : Douleur
Hook B : Demo-first
Hook C : Proof-first
Proof : même clip
CTA : même phrase
```

## Checklist prod hooks (vite mais propre)

- un plan d’ouverture par hook
- une ligne de texte à l’écran (grosse)
- un cut/beat dès la première seconde
- durée stable entre variantes

## Erreurs fréquentes

- 5 changements en même temps (vous n’apprenez rien)
- hooks “clickbait” qui ne matchent pas la proof
- captions illisibles (mobile)
- attendre trop longtemps avant de montrer produit/résultat

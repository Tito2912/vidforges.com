## Comment utiliser cette checklist

Imprimez-la, copiez-la dans Notion, ou transformez-la en tâches. L’idée : supprimer la fatigue de décision.

- vous connaissez toujours la prochaine étape
- vous savez ce que “terminé” veut dire
- vous pouvez déléguer sans tout réexpliquer

Pour le guide pilier complet (map outils + explications), commencez ici : <a href="/fr/guides/creator-stack/">Stack créateur 2026</a>.

## Checklist A — Brief (avant d’écrire)

- Audience et contexte (qui, quand, pourquoi)
- Offre et contraintes (prix, garantie, livraison, mentions)
- Liste de preuves (avis, UGC, stats montrables)
- Un CTA principal (acheter / en savoir plus / DM)
- Un KPI (CTR, CPA, leads, rétention)

Si vous n’arrivez pas à écrire ça en 5 minutes, la vidéo prendra 5 heures.

## Checklist B — Script (simple)

Structure : **Hook → Proof → Demo → CTA**.

- Hook en 1 phrase (pas d’intro)
- Proof que vous pouvez montrer (pas d’invention)
- Demo en séquence de plans (pas en pavés)
- CTA identique à l’écran + à l’oral

## Checklist C — Plans (le levier caché)

Pour une pub 30–45s : **6–12 plans**. Pour un short 15s : **3–6 plans**.

Template :

```
PLAN 01 (0:00–0:02) — Problème + headline
PLAN 02 (0:02–0:06) — Démo 1 (gros plan)
PLAN 03 (0:06–0:10) — Proof (avis / avant-après)
PLAN 04 (0:10–0:18) — Démo 2 (feature / bundle)
PLAN 05 (0:18–0:26) — Offre + prix (texte à l’écran)
PLAN 06 (0:26–0:30) — CTA + urgence + simulation bouton
```

## Checklist D — Pack d’assets (pour ne pas perdre du temps au montage)

Créez un dossier “asset pack” par créa :

- photos produit (PNG transparent si possible)
- logo + couleurs
- screenshots (avis, résultats, UI)
- b‑roll (vos rushs si possible)
- musique (licence OK)

Besoin d’idées b‑roll ? Générez des concepts dans <a href="/fr/tools/kling/">Kling</a>, puis finissez dans un éditeur.

## Checklist E — Draft (template + pacing)

Construisez vite la V1 :

- une seule esthétique de template (pas trois)
- pacing agressif (couper les silences)
- afficher l’offre à l’écran avant 10–15s
- captions intégrées au montage

Les drafts template peuvent être rapides dans <a href="/fr/tools/invideo/">InVideo</a>, mais la finition “brand” reste indispensable.

## Checklist F — Voix + sous‑titres (la clarté d’abord)

En short‑form, les sous‑titres ne sont pas décoratifs : ils font la compréhension.

- style cohérent, fort contraste, safe margins
- texte à l’écran = même offre que l’audio
- voiceover adapté au buyer (ou audio original)

Outils :

- captions + finition : <a href="/fr/tools/capcut/">CapCut</a>
- voix (selon politiques) : <a href="/fr/tools/elevenlabs/">ElevenLabs</a>

## Checklist G — QA (c’est là que vous protégez les winners)

Avant publication :

- La même offre partout (audio, texte, landing)
- Chiffres corrects (prix, remise, livraison, garantie)
- CTA compréhensible sans son
- Conformité (claims, catégories sensibles, règles avant/après)
- Lisibilité mobile (à 30–50% de luminosité)

## Checklist H — Publication (formats + tracking)

- export minimum `9:16` (TikTok/Reels/Shorts)
- 1–2 variantes de caption copy
- UTM ou naming pour tracer les résultats
- frame thumbnail cohérente avec le hook

Template de nommage :

```
ad_<offer>_<angle>__hookA__v03__9x16
ad_<offer>_<angle>__hookB__v03__9x16
ad_<offer>_<angle>__hookA__v03__1x1
```

## Checklist I — Boucle d’itération hebdo (le système qui compound)

1. 1 offre + 1 segment audience pour la semaine.
2. 3–5 hooks (le reste constant).
3. Garder les winners, tuer les losers vite.
4. Une fois le hook gagné : tester proof ou CTA.
5. Documenter les patterns gagnants dans vos templates.

Pour la version “système de tests” : <a href="/fr/guides/ugc-ads-system/">Système UGC Ads</a>.

## Stack recommandée (minimum vs scalable)

Minimum :

- b‑roll : <a href="/fr/tools/kling/">Kling</a>
- captions : <a href="/fr/tools/capcut/">CapCut</a>
- un endroit pour templates + naming (Drive/Notion)

Scalable :

- templates & variantes : <a href="/fr/tools/invideo/">InVideo</a>
- localisation/doublage : <a href="/fr/tools/heygen/">HeyGen</a> ou <a href="/fr/tools/akool/">Akool</a>
- automatisation : <a href="/fr/tools/make/">Make</a>

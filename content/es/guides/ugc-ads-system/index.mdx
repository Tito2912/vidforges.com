## Qué optimiza este sistema (y por qué funciona)

Este sistema está diseñado para 2026: feeds rápidos, usuarios escépticos y plataformas estrictas. Optimiza 4 cosas:

- <strong>Velocidad de aprendizaje</strong>: descubres qué hook/ángulo funciona antes de gastar de más.
- <strong>Consistencia</strong>: puedes producir semanalmente sin “burnout creativo”.
- <strong>Control de variables</strong>: sabes <em>qué cambio</em> movió la métrica (no adivinas).
- <strong>Seguridad</strong>: evitas claims arriesgados y formatos engañosos que destruyen confianza.

## Anatomía en 4 bloques (simple y escalable)

Para anuncios 9:16, esta estructura es difícil de superar:

1. <strong>0–3s Hook</strong>: una promesa o un dolor (con texto en pantalla).
2. <strong>3–10s Prueba</strong>: credibilidad rápida (demo, prueba social, “constraint-based claim”).
3. <strong>10–20s Demo</strong>: enseña el “cómo” con ritmo.
4. <strong>20–30s CTA</strong>: una oferta, una acción.

Tu trabajo es hacer cada bloque <strong>específico</strong>. Lo genérico mata rendimiento.

## Matriz de test (evita el caos)

Empieza con <strong>una oferta</strong> y <strong>una audiencia</strong>. Luego genera variantes con una matriz simple:

- 3 hooks (dolor, resultado, curiosidad)
- 2 pruebas (demo vs prueba social)
- 2 CTA (soft vs directo)

Eso son <strong>12 variantes</strong>. Si vas justo de presupuesto: arranca con <strong>6</strong> (3 hooks × 2 CTA) y añade proof después.

Regla: cuando quieres aprendizaje limpio, cambia <strong>una variable</strong> cada vez.

Ejemplo:

```
Ángulo: “Ahorra tiempo”
Hook A: Dolor
Hook B: Demo-first
Hook C: Proof-first
Proof: mismo clip
CTA: misma frase
```

## Reglas de test limpio (para que el dato sea real)

- Mantén constante la oferta (precio, bundle, garantía).
- Mantén constante la duración (p. ej. 28–32s).
- Cambia solo el inicio cuando testees hooks (primeros 1–3s).
- No mezcles 3 estilos de edición: una estética por “test batch”.
- Guarda el “master” y versiona por mercado/variación (naming).

## Workflow de producción (tool‑agnóstico)

1. Escribe guion base y crea 3 hooks (A/B/C).
2. Prepara un asset pack: 6–12 recursos (clips, capturas, b‑roll, logo).
3. Monta drafts con plantillas (<a href="/es/tools/invideo/">InVideo</a> es ideal para variantes).
4. Acaba en un editor (<a href="/es/tools/capcut/">CapCut</a>): subtítulos + ritmo + export.
5. Añade voz si hace falta (<a href="/es/tools/elevenlabs/">ElevenLabs</a>) y mantenla corta.
6. Exporta en ratios correctos y guarda variantes con naming consistente.

Si necesitas b‑roll generado: <a href="/es/tools/kling/">Kling</a>. Para presentador/localización: <a href="/es/tools/heygen/">HeyGen</a> / <a href="/es/tools/akool/">Akool</a>.

## Naming y tracking (no negociable)

Sin naming, no puedes aprender. Plantilla simple:

```
ad_<offer>_<angle>__hookA__proofDemo__ctaSoft__v01__9x16
ad_<offer>_<angle>__hookB__proofSocial__ctaDirect__v01__9x16
```

Registra en una hoja (o Notion):

- fecha, plataforma, campaña/adset
- variante exacta (hook/proof/cta)
- métricas clave + nota de observación (“drop a los 6s”, “CTA confuso”)

## Métricas clave (y cómo interpretarlas)

- <strong>Thumbstop / 2s hold</strong>: si es bajo, el problema suele ser el hook (promesa vaga o plano débil).
- <strong>Retención</strong>: caída en mitad = ritmo/claridad; caída al final = CTA/oferta.
- <strong>CTR</strong>: si la gente mira pero no hace click, suele ser oferta o proof.
- <strong>CPA / ROAS</strong>: valida si el click es de calidad (no solo “curiosidad”).

## Cadencia semanal (para que el sistema componga)

1. Elige 1 oferta + 1 ángulo para la semana.
2. Produce 3–5 hooks (resto constante).
3. Lanza, elige winner rápido.
4. Solo después: testea proof o CTA.
5. Documenta patrones ganadores (copy, planos, captions) para reutilizar.

## Escalar winners sin romper lo que funciona

Una vez tengas un hook ganador:

- duplica el creative y prueba 1 proof alternativo (demo vs social proof)
- prueba 1 CTA alternativo (soft vs directo)
- crea una variante más corta (15s) <em>sin cambiar el hook</em>
- localiza la versión ganadora (subtítulos → doblaje) con la guía de <a href="/es/guides/video-localization/">Localización de vídeo</a>

## Compliance / brand safety (realidad 2026)

- Evita suplantación engañosa o formatos tipo “falso portavoz”.
- Usa caras/voces con consentimiento y assets licenciados.
- Claims verificables + aclaraciones cuando aplica (“terms apply”, “results vary”).
- Si dudas: transparencia > “trucos” creativos. La confianza compone.

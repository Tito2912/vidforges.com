## Regla: hooks primero

Si el hook es débil, el montaje no lo arregla. Arregla los 3 primeros segundos:

- texto
- plano inicial
- promesa/proof

Sistema pilar: <a href="/es/guides/ugc-ads-system/">UGC Ads System</a>.

## Categorías de hooks

| Categoría | Patrón | Ejemplo (plantilla) |
|---|---|---|
| Dolor → solución | Problema directo | “Si sigues haciendo **X**, pierdes **Y**…” |
| Verdad inesperada | Rompe creencia | “Todos dicen **X**. Es falso. Mira…” |
| Proof-first | Prueba al inicio | “Lo medí 7 días… resultado:” |
| Demo-first | Acción inmediata | “Mira esto—la forma más rápida de…” |
| Error | Enseña sin juzgar | “Deja de hacer **X**—haz **Y**.” |
| Comparación | A vs B | “Probé **A vs B**… y ganó…” |
| Oferta | Deal upfront | “Hoy: **-X%** + envío gratis.” |
| Historia | Micro story | “Casi lo dejo… hasta que probé esto.” |

## Subtítulos y texto en pantalla

Reglas:

- una idea por línea
- alto contraste y márgenes seguros
- deja el hook on-screen 1–2s más
- oferta consistente en audio y texto

Stack:

- subtítulos + polish: <a href="/es/tools/capcut/">CapCut</a>
- variantes desde plantillas: <a href="/es/tools/invideo/">InVideo</a>
- b‑roll rápido: <a href="/es/tools/kling/">Kling</a>

## Sistema de testing limpio

1. Mismo offer, proof, CTA y duración.
2. Cambia solo los 1–3 segundos iniciales.
3. 3–5 hooks por ángulo.
4. Winner → luego test de proof o CTA.

```
Ángulo: "Ahorra tiempo"
Hook A: Dolor
Hook B: Demo-first
Hook C: Proof-first
```

## Errores comunes

- Cambiar 5 cosas a la vez
- Clickbait que no conecta con proof
- Texto pequeño (móvil)
- Tardar demasiado en mostrar resultado

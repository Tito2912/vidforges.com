## Cómo usar esta checklist

Imprímela, pásala a Notion o conviértela en tareas. La idea es eliminar decisiones repetidas:

- siempre sabes el siguiente paso
- siempre sabes qué significa “listo”
- puedes delegar sin reexplicar

Para la guía pilar (mapa de herramientas + más detalle): <a href="/es/guides/creator-stack/">Creator Stack 2026</a>.

## Checklist A — Brief

- Audiencia y contexto (quién, cuándo, por qué)
- Oferta y restricciones (precio, garantía, envío, legales)
- Lista de proof (reseñas, UGC, stats que puedas mostrar)
- Un CTA principal
- Un KPI (CTR, CPA, leads, retención)

## Checklist B — Guion

Estructura: **Hook → Proof → Demo → CTA**.

- Hook en 1 frase
- Proof real (no inventar)
- Demo como secuencia de planos
- CTA idéntico en audio y en pantalla

## Checklist C — Planos

Para un ad de 30–45s: **6–12 planos**. Para 15s: **3–6 planos**.

```
PLANO 01 (0:00–0:02) — Problema + headline
PLANO 02 (0:02–0:06) — Demo 1 (close-up)
PLANO 03 (0:06–0:10) — Proof (reseña / antes-después)
PLANO 04 (0:10–0:18) — Demo 2 (feature / bundle)
PLANO 05 (0:18–0:26) — Oferta + precio (texto)
PLANO 06 (0:26–0:30) — CTA + urgencia + botón simulado
```

## Checklist D — Asset pack

Una carpeta por creatividad:

- fotos de producto
- logo + colores
- screenshots (reviews, resultados, UI)
- b‑roll (mejor si es propio)
- música con licencia

Para ideas de b‑roll: <a href="/es/tools/kling/">Kling</a>.

## Checklist E — Draft (plantilla + pacing)

- una estética por plantilla
- recorta silencios
- oferta on-screen antes de 10–15s
- subtítulos como parte del montaje

Drafts rápidos con <a href="/es/tools/invideo/">InVideo</a>, y remate final siempre a tu estilo.

## Checklist F — Voz + subtítulos

Herramientas:

- subtítulos + polish: <a href="/es/tools/capcut/">CapCut</a>
- voiceovers (si aplica): <a href="/es/tools/elevenlabs/">ElevenLabs</a>

## Checklist G — QA

- oferta consistente (audio, texto, landing)
- números correctos
- CTA entendible sin sonido
- compliance de claims
- legibilidad móvil

## Checklist H — Publicación

- export `9:16`
- 1–2 variantes de copy
- naming/UTM para medir

```
ad_<offer>_<angle>__hookA__v03__9x16
ad_<offer>_<angle>__hookB__v03__9x16
```

## Checklist I — Iteración semanal

1. 1 oferta + 1 segmento.
2. 3–5 hooks (resto constante).
3. winners se quedan, losers se eliminan.
4. luego test de proof o CTA.
5. documenta patrones ganadores.

Sistema de tests: <a href="/es/guides/ugc-ads-system/">UGC Ads System</a>.

## Stack recomendada

Mínimo:

- b‑roll: <a href="/es/tools/kling/">Kling</a>
- subtítulos: <a href="/es/tools/capcut/">CapCut</a>

Escalable:

- variantes: <a href="/es/tools/invideo/">InVideo</a>
- localización/doblaje: <a href="/es/tools/heygen/">HeyGen</a> o <a href="/es/tools/akool/">Akool</a>
- automatización: <a href="/es/tools/make/">Make</a>

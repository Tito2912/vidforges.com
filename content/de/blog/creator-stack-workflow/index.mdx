## So nutzt du die Checklist

Print, Notion, Task-Tool—egal. Ziel: weniger Entscheidungen, mehr Output.

Für den Pillar Guide (Tool Map + Tiefe): <a href="/de/guides/creator-stack/">Creator Stack 2026</a>.

## Checklist A — Brief

- Audience + Kontext
- Offer + Constraints (Preis, Garantie, Legal Lines)
- Proof (Reviews, UGC Clips, Stats)
- 1 CTA
- 1 KPI (CTR, CPA, Leads, Retention)

## Checklist B — Script

Struktur: **Hook → Proof → Demo → CTA**.

- Hook: 1 Satz
- Proof: zeigbar, nicht erfunden
- Demo: als Shot‑Sequence
- CTA: identisch on-screen + spoken

## Checklist C — Shots

30–45s Ad: **6–12 Shots**. 15s Short: **3–6 Shots**.

```
SHOT 01 (0:00–0:02) — Problem + Headline
SHOT 02 (0:02–0:06) — Demo 1 (close-up)
SHOT 03 (0:06–0:10) — Proof (Review / Before-After)
SHOT 04 (0:10–0:18) — Demo 2 (Feature / Bundle)
SHOT 05 (0:18–0:26) — Offer + Price (on-screen)
SHOT 06 (0:26–0:30) — CTA + Urgency + Button-Sim
```

## Checklist D — Asset Pack

- Product Photos
- Logo + Brand Colors
- Screenshots (Reviews, Results, UI)
- b‑roll (own footage when possible)
- licensed music

b‑roll Ideen: <a href="/de/tools/kling/">Kling</a>.

## Checklist E — Draft (Template + Pacing)

Template Drafts können schnell in <a href="/de/tools/invideo/">InVideo</a> entstehen—finish aber immer brand‑konform.

## Checklist F — Voice + Captions

- Captions sind Comprehension
- On-screen Offer muss exakt passen

Tools:

- Captions + Finish: <a href="/de/tools/capcut/">CapCut</a>
- Voiceover (wenn erlaubt): <a href="/de/tools/elevenlabs/">ElevenLabs</a>

## Checklist G — QA

- Offer überall gleich (Audio/Text/Landing)
- Zahlen korrekt
- CTA ohne Sound verständlich
- Claims compliant
- Mobile readability

## Checklist H — Publish

- Export `9:16`
- 1–2 Copy-Varianten
- Naming/UTM für Tracking

```
ad_<offer>_<angle>__hookA__v03__9x16
ad_<offer>_<angle>__hookB__v03__9x16
```

## Checklist I — Weekly Iteration Loop

1. 1 Offer + 1 Audience Segment.
2. 3–5 Hook Varianten.
3. Winners behalten, Losers killen.
4. Danach Proof oder CTA testen.
5. Winning Patterns dokumentieren.

Testing System: <a href="/de/guides/ugc-ads-system/">UGC Ads System</a>.
